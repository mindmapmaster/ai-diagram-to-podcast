## 识别到的节点与连接

### 1. 起点与登录阶段
| 节点 | 类型 | 说明 |
|------|------|------|
| 进入系统首页 | 起止框 | 流程入口 |
| 判断用户名和密码 | 判断菱形 | 关键分支点：正确 / 错误 |
| 提示用户名或密码错误，重新输入 | 处理框 | 错误分支的反馈动作 |

**连接关系**：
- 进入系统首页 → 判断用户名和密码
- 判断用户名和密码（错误） → 提示用户名或密码错误，重新输入 → 回到判断用户名和密码（循环）
- 判断用户名和密码（正确） → 进入图书借阅系统

### 2. 主功能选择阶段
| 节点 | 类型 | 说明 |
|------|------|------|
| 进入图书借阅系统 | 处理框 | 登录成功后的主入口 |
| 浏览图书信息 | 处理框 | 功能分支 A |
| 借阅图书 | 处理框 | 功能分支 B |
| 查看借阅记录 | 处理框 | 功能分支 C |

**连接关系**：
- 进入图书借阅系统 → 三条并行分支：
  - 浏览图书信息
  - 借阅图书
  - 查看借阅记录

### 3. 借阅子流程阶段
| 节点 | 类型 | 说明 |
|------|------|------|
| 确认办理借阅？ | 判断菱形 | 用户确认分支 |
| 登记借阅信息，设置归还期限 | 处理框 | 借阅确认后的核心动作 |

**连接关系**：
- 借阅图书 → 确认办理借阅？
- 确认办理借阅？（否） → 回到进入图书借阅系统（可重新选择）
- 确认办理借阅？（是） → 登记借阅信息，设置归还期限

### 4. 归还与逾期处理阶段
| 节点 | 类型 | 说明 |
|------|------|------|
| 借阅期限到期 | 处理框 | 触发条件 |
| 超期提醒，缴纳逾期费用 | 处理框 | 逾期分支 |
| 读者主动归还图书 | 处理框 | 正常归还分支 |
| 完成归还，更新借阅状态 | 处理框 | 正常归还后的动作 |
| 借阅流程结束 | 处理框 | 借阅子流程终点 |

**连接关系**：
- 登记借阅信息，设置归还期限 → 两条并行触发路径：
  - 借阅期限到期 → 超期提醒，缴纳逾期费用 → 借阅流程结束
  - 读者主动归还图书 → 完成归还，更新借阅状态 → 借阅流程结束

### 5. 退出阶段
| 节点 | 类型 | 说明 |
|------|------|------|
| 退出系统 | 起止框 | 整个业务流程终点 |

**连接关系**：
- 浏览图书信息 → 选择退出 → 退出系统
- 查看借阅记录 → 选择退出 → 退出系统
- 借阅流程结束 → 退出系统

## 流程逻辑特点

1. **循环结构**：用户名/密码错误后允许重新输入，形成登录循环。
2. **多分支并行**：登录成功后提供 3 个独立功能入口（浏览、借阅、查记录）。
3. **判断分支**：两次菱形判断——登录验证、借阅确认。
4. **合并节点**：浏览/查记录通过「选择退出」合并到退出系统；借阅通过「借阅流程结束」合并到退出系统。
5. **异常处理**：超期归还作为独立分支存在，说明系统有逾期费用机制。
